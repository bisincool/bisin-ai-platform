bisincool.co.jp（合同会社BISIN）向けに設計・実装したCOOLSPA導入相談チャットウィジェットの完全記録。次のB2Bサイト制作に流用可能なテンプレートとして整理。

ファイル構成

project/ ├── css/ │ └── chatbot.css # ウィジェット専用スタイル（既存style.cssと完全分離） ├── js/ │ └── chatbot.js # 自己完結型IIFE、外部依存ゼロ ├── index.html ├── about.html # 全ページ共通で以下を </body> 直前に挿入 └── ...
各HTMLページへの挿入（</body>直前）
<link rel="stylesheet" href="css/chatbot.css" />
<script src="js/chatbot.js"></script>
⚠️ 必ず </body> 直前に置くこと。<head>内や他スクリプトの前に置くと、DOMが未構築でwidgetが生成できない。chatbot.js内でDOMContentLoadedを使って対応済みだが、bodyの末尾が最も安全。

el() ヘルパー — 最重要バグ修正

🚨 このバグを見落とすと全CSS クラスが無効化され、チャット全体が画面左端に流れ全幅表示になる。必ず修正済みバージョンを使うこと。
❌ バグあり（よくある間違い）
function el(tag, attrs) {
  var node = document.createElement(tag);
  Object.keys(attrs || {}).forEach(function(k) {
    node.setAttribute(k, attrs[k]);   // ← setAttribute('className', ...) は
  });                                 //   HTML属性 classname= を作るだけ！
  return node;                        //   CSSのクラスセレクタが当たらない
}
✅ 修正済み（こちらを使う）
function el(tag, attrs) {
  var node = document.createElement(tag);
  Object.keys(attrs || {}).forEach(function(k) {
    if (k === 'className') { node.className = attrs[k]; }  // ← DOM プロパティで設定
    else { node.setAttribute(k, attrs[k]); }
  });
  return node;
}
方法結果CSS有効？
node.setAttribute('className', 'cb-launcher') HTML属性 classname="cb-launcher"（小文字）が付く ❌ 無効
node.setAttribute('class', 'cb-launcher') 正しい class="cb-launcher" 属性 ✅ 有効
node.className = 'cb-launcher' DOMプロパティ経由で正しく設定 ✅ 有効（推奨）

フォーム送信 — mailto 方式

✅ Formspree・サーバー不要。静的ホスティング（Cloudflare Pages / GitHub Pages）で完全動作。
mailto URL の組み立て
function openMailto() {
  var bodyText = [
    '■ お客様情報',
    'お名前　　　: ' + name,
    'サロン名　　: ' + salon,
    'メール　　　: ' + email,
    '電話番号　　: ' + (tel || '未記入'),
    '',
    '■ チャットで閲覧されたトピック',
    'トピック　　: ' + (answers.topic || '未選択'),
    '',
    '■ ご質問・ご要望',
    msg || '（なし）',
  ].join('\n');

  var subject   = encodeURIComponent('【COOLSPA導入相談】' + salon);
  var mailto    = 'mailto:bisin202603@gmail.com?subject='
                + subject + '&body=' + encodeURIComponent(bodyText);

  var link = document.createElement('a');
  link.href = mailto; link.style.display = 'none';
  document.body.appendChild(link);
  link.click();
  document.body.removeChild(link);

  setTimeout(showSuccess, 600);   // 600ms後に完了画面へ
}

バグ一覧 & 対処法

症状原因修正
チャットが左端・全幅表示 el()のsetAttribute('className')がclassname=属性を作りCSSクラス無効化 → position:staticになる node.className = attrs[k] に変更
修正ファイルを保存してもブラウザに反映されない ブラウザが<script src="chatbot.js">をキャッシュしている デバッグ時は src='chatbot.js?' + Date.now() でキャッシュバスト
git commitしても本番に反映されない worktreeで編集したがmainブランチのファイルが別 mainディレクトリで直接編集 or worktreeブランチをmainにmerge
position:fixedが効かず画面外に出る 親要素にtransform/filter/will-changeがあると新たなcontaining blockが作られる particles.jsなど画面エフェクトを確認。要素を<html>直下に追加するか対象要素のスタイルを調査
worktreeのファイルとmainが食い違う worktreeで修正 → mainに反映せず直接pushした 両方に同じ修正を適用するか、worktreeのブランチをmergeしてから push

position:fixed が崩れる原因

⚠️ 以下のCSSプロパティが祖先要素に付いていると、position:fixedの基準がビューポートではなくその要素になる（＝固定にならない）。
プロパティ影響よくある出どころ
transform: *fixed の基準がその要素になるparticles.js, スクロールアニメーション
filter: *同上背景ぼかし、オーバーレイ効果
will-change: transform同上パフォーマンス最適化記述
overflow: hidden on body一部ブラウザでfixed挙動が変わる横スクロール防止のよくある記述
perspective: *3D文脈でfixedが崩れる3Dアニメーション
